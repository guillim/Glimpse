Design — Section 2: Session Monitoring

How we detect active sessions and determine what each one is doing

Session Discovery

File System Scan (every 5 seconds)
~/.claude/projects/
  ├── -Users-gui-github-background/
  │     ├── c9555423-....jsonl   ← session log
  │     └── 161fee38-.../
  │           └── subagents/     ← ignored (sub-agents not separate characters)
  ├── -Users-gui-github-paul/
  │     └── b0dfe572-....jsonl   ← another session
  └── ...

Discovery algorithm:
1. Glob all *.jsonl files in ~/.claude/projects/*/ (depth 1 only)
2. For each, read last modified time
3. Active = modified in last 5 minutes
4. Stale = modified 5-10 min ago → fade out candidate
5. Dead = modified 10+ min ago → ignore

Activity Detection

Read last ~10 lines of each active JSONL to classify current state

Activity State Machine
📖 READING
  Last assistant message contains tool_use with name in:
  ["Read", "Glob", "Grep", "Bash(cat/head/tail)"]
  → Character holds a book

💬 TALKING
  Last assistant message contains "text" content block
  (model is outputting a response to the user)
  → Character has speech bubble

❓ WAITING
  Last message type = "assistant" with stop_reason = "end_turn"
  AND no new messages for 30+ seconds
  (session is waiting for human input)
  → Character has question mark

💤 SLEEPING
  Default / fallback state
  Session is active but no recent activity (>2 min since last message)
  → Character has zzz animation

Topic Detection

The word below each character describing what the session is about

Topic Extraction
Strategy: Use the project directory name + last user message

1. Project name from path:
   "-Users-gui-github-background" → "background"
   "-Users-gui-github-paul"      → "paul"

2. This becomes the label below the character

Simple, reliable, no AI needed.
The project name tells you which codebase the agent is working on.

Session Lifecycle

New session appears CharacterGenerator creates unique creature from session ID. Character fades in over 0.5s. Grid relayouts with animation.
Session active Status icon updates every 5s poll. Smooth transitions between states (book → speech bubble, etc).
Session ends "Goodbye!" text appears above character for 5 seconds, then character fades out over 1s. Grid relayouts.
No sessions Centered text on dark background: "No Claude sessions active — start one to see your Pokemon!"
