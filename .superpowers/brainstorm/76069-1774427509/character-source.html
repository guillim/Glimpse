Character Asset Source

How should we get the pixel art Pokemon sprites into the app?

A

Bundled PNG Sprites

Ship pre-made pixel art PNGs in the app bundle. Each character is a sprite sheet with idle + status variants (reading, talking, sleeping, question). I'd create these as simple colored shapes in code or you'd source them externally.

Pros

  • Fast rendering, no generation overhead
  • Pixel-perfect control
  • Zero network dependency

Cons

  • Need to create/find ~10+ sprite sheets
  • Adding new characters = new assets
B

Procedurally Generated in Code

Draw characters programmatically using Core Graphics — colored shapes, eyes, features built from primitives. Each session gets a unique character generated from a seed (session ID). Pokemon-styled but original.

Pros

  • Infinite unique characters
  • No external assets needed
  • Tiny memory footprint

Cons

  • Won't look like actual Pokemon
  • More complex code
  • Limited artistic quality
C

Hybrid: Bundled Base + Code Overlays

Bundle a set of base character PNGs (Pikachu, Charmander, Bulbasaur, etc.) and draw status indicators (book, speech bubble, zzz, question mark) programmatically as overlays. Characters cycle through the roster as sessions appear.

Pros

  • Real Pokemon look for base characters
  • Status overlays are flexible and lightweight
  • Easy to add more characters later

Cons

  • Need to source/create base sprites
  • Limited roster (but expandable)

Recommendation: C (Hybrid) — real Pokemon base sprites give the authentic feel, while code-drawn overlays keep status indicators flexible without needing separate sprite sheets for every state.
