Architecture Approaches

Three ways to build the Pokemon session monitor theme. All share the same features — they differ in rendering strategy.

A

SpriteKit Scene (Recommended)

Add the theme as an SKScene rendered inside an SKView, replacing the SCNView when this theme is active. SpriteKit is Apple's 2D engine — purpose-built for sprites, text labels, and 2D animations.

Pros

  • Lowest RAM — no 3D pipeline, no depth buffer, no lighting
  • Native sprite nodes, text nodes, animation actions
  • Built-in physics-free layout (no SceneKit overhead)
  • GPU-accelerated but lightweight
  • ~60fps trivially for 2D sprites

Cons

  • New view type — need to swap SCNView ↔ SKView when switching themes
  • Head tracking integration needs adapter (currently tied to SCNSceneRendererDelegate)
B

SceneKit with Flat Planes

Stay in SceneKit — render characters as SCNPlane nodes with CGImage textures. Draw sprites to bitmaps via Core Graphics, apply as materials.

Pros

  • No view swapping — same SCNView for all themes
  • Head tracking works as-is
  • Can still use 3D camera for subtle parallax on sprites

Cons

  • More RAM — full 3D pipeline running for 2D content
  • Text rendering via SCNText or texture bitmaps (clunky)
  • Animations need manual SCNAction choreography
  • Depth buffer, lighting pipeline all running unnecessarily
C

Core Animation (CALayer)

Skip game engines entirely. Use CALayer sublayers with Core Graphics–drawn images. Animate with CABasicAnimation. Lightest possible approach.

Pros

  • Absolute minimum RAM — no game engine at all
  • Native text rendering (CATextLayer)
  • Simplest code for static + simple animations

Cons

  • No game loop — timer-based updates feel janky for continuous animation
  • Complex animations are painful in CA
  • Head tracking per-frame updates fight CA's implicit animation model
  • Completely different rendering path from all other themes

Recommendation: A (SpriteKit) — It's the right tool for 2D sprites. The view-swapping cost is a one-time engineering task, and the RAM/performance benefits over SceneKit-for-2D are significant. CALayer is tempting but breaks down for continuous head-tracking-driven updates.
