State-Specific Animations — Approaches

Each replaces the default breathing. Subtle, ambient feel.

A

Distinct Motion Per State

Each state gets a unique motion type: sway for thinking, bounce for running, jitter for writing, etc. Easy to tell apart even in peripheral vision because the kind of movement differs.

🐣spawn

✅done
❓ask

Pros

  • Maximum differentiation — each state is visually unique
  • 9 animations but all are simple SKAction sequences

Cons

  • More animation code to maintain
  • Some motions (jitter for writing) might feel noisy with many agents
B

Speed + Rhythm Variation

All states use similar bob/breathe motions but with different speeds and rhythms. Running is fast and sharp, thinking is slow and floaty, sleeping is barely moving. Calmer overall but relies more on tempo than motion type.

Pros

  • Calmer scene — motions harmonize rather than compete
  • Easier to tune (just adjust timing parameters)

Cons

  • Harder to distinguish states at a glance — speed is subtler than motion type
  • May lose differentiation with many agents on screen
